*   **קובץ App.js:**
```javascript
import React, { useState, useEffect } from 'react';
import { auth, db } from './firebase';
import { collection, addDoc, orderBy, query, onSnapshot } from 'firebase/firestore';
import { useAuthState } from 'react-firebase-hooks';

function App() {
  const [user] = useAuthState(auth);
  const [messages, setMessages] = useState([]);
  const [newMessage, setNewMessage] = useState('');

  useEffect(() => {
    if (user) {
      const messagesRef = collection(db, 'messages');
      const q = query(messagesRef, orderBy('createdAt'));
      const unsubscribe = onSnapshot(q, (snapshot) => {
        setMessages(snapshot.docs.map((doc) => ({
          id: doc.id,
          text: doc.data().text,
          createdAt: doc.data().createdAt,
          senderId: doc.data().senderId
        })));
      });
      return () => unsubscribe();
    }
  }, [user]);

  const sendMessage = async (text) => {
    if (text.trim() !== '') {
      const messagesRef = collection(db, 'messages');
      await addDoc(messagesRef, {
        text: text,
        createdAt: new Date(),
        senderId: user.uid
      });
      setNewMessage('');
    }
  };

  return (
    <div className="App">
      <header className="App-header">
        <h1>Chat App</h1>
        {user ? (
          <p>Logged in as: {user.email}</p>
        ) : (
          <button onClick={() => auth.signInWithEmailAndPassword("test@example.com", "password")}>Sign In</button>
        )}
      </header>
      <div className="chat-container">
        <ul>
          {messages.map((message) => (
            <li key={message.id} className={message.senderId === user?.uid ? 'sent' : 'received'}>
              {message.text}
              <span className="timestamp">{new Date(message.createdAt).toLocaleTimeString()}</span>
            </li>
          ))}
        </ul>
      </div>
      <div className="input-container">
        <input
          type="text"
          value={newMessage}
          onChange={(e) => setNewMessage(e.target.value)}
          placeholder="Type your message..."
          onKeyPress={(e) => {
            if (e.key === 'Enter') {
              sendMessage(newMessage);
            }
          }}
        />
        <button onClick={() => sendMessage(newMessage)}>Send</button>
      </div>
    </div>
  );
}

export default App;
```

**מהו Regularization?**

Regularization היא טכניקה המשמשת בלמידת מכונה כדי למנוע overfitting (התאמת יתר) – מצב שבו מודל לומד את נתוני האימון בצורה מושלמת מדי, אך מתקשה לבצע תחזיות טובות על נתונים חדשים. Regularization מוסיפה "עונש" לפונקציית העלות, שמכריחה את המודל להיות פשוט יותר.

**1. L1 Regularization (Lasso)**

*   **עיקרון:** מוסיפה עונש שווה לסכום הערכים המוחלטים של המקדמים (coefficients).
*   **נוסחה:**
    *   פונקציית העלות המקורית: J(w)
    *   פונקציית העלות עם L1 regularization: J(w) + λ * Σ |wi|
    *   כאשר:
        *   w הם המקדמים של המודל
        *   λ (למדה) הוא פרמטר ה-regularization (ככל שהוא גבוה יותר, העונש על מקדמים גדולים יותר גדל)
        *   Σ |wi| הוא סכום הערכים המוחלטים של המקדמים.
*   **דוגמת קוד בפייתון (scikit-learn):**

```python
from sklearn.linear_model import Lasso
from sklearn.preprocessing import StandardScaler
import numpy as np

# נתונים לדוגמה
X = np.array([[1, 2], [3, 4], [5, 6], [7, 8]])
y = np.array([10, 12, 14, 16])

# סקיילינג של הנתונים
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# יצירת מודל Lasso
lasso = Lasso(alpha=0.5)  # alpha הוא פרמטר ה-regularization (λ)
lasso.fit(X_scaled, y)

# הדפסת המקדמים
print("Mקדמים:", lasso.coef_)
```

*   **יתרונות:**
    *   **Feature Selection:** L1 regularization יכולה להביא לחלק מהמקדמים להיות בדיוק 0, מה שמאפשר לבצע feature selection אוטומטית ולהסיר תכונות לא רלוונטיות.
    *   **Sparse Model:** יוצרת מודלים "דלילים" עם מעט מקדמים שונים מאפס.
*   **חסרונות:**
    *   **יכולה להיות מוטה:** בגלל האופי של ה-regularization, היא עלולה להוביל למקדמים מוטים, במיוחד אם יש תכונות חשובות.

**2. L2 Regularization (Ridge)**

*   **עיקרון:** מוסיפה עונש שווה לריבוע של המקדמים.
*   **נוסחה:**
    *   פונקציית העלות המקורית: J(w)
    *   פונקציית העלות עם L2 regularization: J(w) + λ * Σ wi²
    *   כאשר:
        *   w הם המקדמים של המודל
        *   λ (למדה) הוא פרמטר ה-regularization
        *   Σ wi² הוא סכום הריבועים של המקדמים.
*   **דוגמת קוד בפייתון (scikit-learn):**

```python
from sklearn.linear_model import Ridge
from sklearn.preprocessing import StandardScaler
import numpy as np

# נתונים לדוגמה (כמו קודם)
X = np.array([[1, 2], [3, 4], [5, 6], [7, 8]])
y = np.array([10, 12, 14, 16])

# סקיילינג של הנתונים
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# יצירת מודל Ridge
ridge = Ridge(alpha=0.5)  # alpha הוא פרמטר ה-regularization (λ)
ridge.fit(X_scaled, y)

# הדפסת המקדמים
print("Mקדמים:", ridge.coef_)
```

*   **יתרונות:**
    *   **Stable Solution:**  מובילה לפתרון יציב יותר, במיוחד כאשר יש מתאם גבוה בין התכונות.
    *   **Reduces Variance:** מצמצמת את השונות של המודל, מה שמביא לביצועים טובים יותר על נתונים חדשים.
*   **חסרונות:**
    *   **לא מבצעת Feature Selection:** לא גורמת למקדמים להיות בדיוק 0, ולכן לא מסירה תכונות לא רלוונטיות.
    *   **יכולה להיות פחות יעילה מ-L1 במקרים מסוימים.**

**3. Elastic Net Regularization**

*   **עיקרון:** שילוב של L1 ו-L2 regularization.
*   **נוסחה:**
    *   פונקציית העלות עם Elastic Net regularization: J(w) + λ1 * Σ |wi| + λ2 * Σ wi²
    *   כאשר:
        *   λ1 ו-λ2 הם פרמטרים ששולטים בעוצמת ה-L1 ו-L2 regularization, בהתאמה.
*   **דוגמת קוד בפייתון (scikit-learn):**

```python
from sklearn.linear_model import ElasticNet
from sklearn.preprocessing import StandardScaler
import numpy as np

# נתונים לדוגמה
X = np.array([[1, 2], [3, 4], [5, 6], [7, 8]])
y = np.array([10, 12, 14, 16])

# סקיילינג של הנתונים
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# יצירת מודל ElasticNet
elastic_net = ElasticNet(alpha=0.5, l1_ratio=0.5)  # alpha הוא סך העונש, l1_ratio קובע את האיזון בין L1 ו-L2
elastic_net.fit(X_scaled, y)

# הדפסת המקדמים
print("Mקדמים:", elastic_net.coef_)
```

*   **יתרונות:**
    *   **משלבת את היתרונות של L1 ו-L2:**  מבצעת feature selection (כמו L1) ומצמצמת את השונות (כמו L2).
    *   **יכולה להיות יעילה יותר מ-L1 או L2 בנפרד.**
*   **חסרונות:**
    *   **מורכבת יותר לכוונון:** דורשת כוונון שני פרמטרים (α ו-l1_ratio).

**4. Dropout (בנויבנט)**

*   **עיקרון:** טכניקה regularization ספציפית לרשתות נוירונים. במהלך האימון, באופן אקראי, חלק מהנוירונים מוחלשים או מוסרים (dropout).
*   **יתרונות:**
    *   **מונעת תלות יתר בנוירונים ספציפיים:** גורמת לרשת ללמוד תכונות חזקות יותר.
    *   **קלות יישום:** קל מאוד להוסיף dropout לרשת נוירונים.
*   **חסרונות:**
    *   **יכולה להאריך את זמן האימון.**

**בחירת סוג Regularization:**

*   **L1:** מתאים למקרים שבהם רוצים לבצע feature selection אוטומטית.
*   **L2:** מתאים למקרים שבהם רוצים לצמצם את השונות של המודל.
*   **Elastic Net:** מתאים למקרים שבהם רוצים לשלב את היתרונות של L1 ו-L2.
*   **Dropout:** מתאים לרשתות נוירונים.

חשוב להתנסות עם סוגי regularization שונים ולבחור את השיטה שמניבה את הביצועים הטובים ביותר על נתוני האימות או בדיקה.
