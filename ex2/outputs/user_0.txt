**שלב 2: אתחול Firebase**

1.  **הגדרת Firebase:** ב-Firebase Console, עבור ל"Project settings" > "Service accounts". הורד את קובץ `firebase-config.js` ושמור אותו בתיקייה `src`.
2.  **אתחול Firebase ב-App.js:**

    ```javascript
    // src/App.js
    import React, { useState, useEffect } from 'react';
    import { initializeApp } from 'firebase/app';
    import { getFirestore, collection, addDoc, onSnapshot, orderBy, query } from 'firebase/firestore';

    const firebaseConfig = import('./firebase-config.js');
    const app = initializeApp(firebaseConfig);
    const db = getFirestore(app);

    function App() {
      const [messages, setMessages] = useState([]);
      const [newMessage, setNewMessage] = useState('');
      const [user, setUser] = useState('User'); // שם משתמש ברירת מחדל

      useEffect(() => {
        const messagesRef = collection(db, 'messages');
        const q = query(messagesRef, orderBy('createdAt'));

        const unsubscribe = onSnapshot(q, (snapshot) => {
          const newMessages = snapshot.docs.map((doc) => ({
            id: doc.id,
            ...doc.data(),
          }));
          setMessages(newMessages);
        });

        return unsubscribe;
      }, [db]);

      const sendMessage = async () => {
        const messagesRef = collection(db, 'messages');
        const now = new Date();
        await addDoc(messagesRef, {
          text: newMessage,
          user: user,
          createdAt: now,
        });
        setNewMessage('');
      };

      return (
        <div className="App">
          <h1>React Chat App with Firebase</h1>
          <div className="chat-container">
            {messages.map((message) => (
              <div key={message.id} className="message">
                <span>{message.user}:</span>
                <span>{message.text}</span>
                <small>{message.createdAt.toLocaleTimeString()}</small>
              </div>
            ))}
          </div>
          <div className="input-container">
            <input
              type="text"
              value={newMessage}
              onChange={(e) => setNewMessage(e.target.value)}
              placeholder="Type your message..."
            />
            <button onClick={sendMessage}>Send</button>
            <input
              type="text"
              value={user}
              onChange={(e) => setUser(e.target.value)}
              placeholder="Your Name"
            />
          </div>
        </div>
      );
    }

    export default App;
    ```

Prompt: תרחיב על כל סוגי ה-regularization בלמידת מכונה, כולל נוסחאות, דוגמאות קוד, ויתרונות וחסרונות

בסדר גמור, הנה סקירה מפורטת על סוגי ה-regularization השונים בלמידת מכונה, כולל נוסחאות, דוגמאות קוד בפייתון עם scikit-learn, ודיון ביתרונות ובחסרונות של כל שיטה.

**מהו Regularization?**

Regularization היא טכניקה המשמשת כדי למנוע התאמת יתר (Overfitting) במודלים של למידת מכונה. התאמת יתר מתרחשת כאשר מודל לומד את נתוני האימון בצורה טובה מדי, כולל רעש וערכים חריגים, וכתוצאה מכך מתקשה להכליל נכון לנתונים חדשים. Regularization מוסיפה "עונש" למודל על מורכבות יתר, ומעודדת מודלים פשוטים יותר.

**סוגי Regularization:**

1. **L1 Regularization (Lasso Regression)**

*   **עקרון:** מוסיפה "עונש" על הסכום המוחלט של המקדמים (weights) של המודל.
*   **נוסחה:**
    *   Cost Function = Loss Function + λ * Σ |βᵢ|  (כאשר λ הוא פרמטר הרגולריזציה, ו-βᵢ הם המקדמים)
*   **יתרונות:**
    *   **Feature Selection:**  L1 יכול "להרוג" מקדמים של תכונות מסוימות, כלומר להפוך אותם לאפס, ובכך לבצע אוטומטית בחירת תכונות (Feature Selection).
    *   **מודלים דלילים:**  יוצרת מודלים דלילים (Sparse Models), כלומר מודלים עם מעט מקדמים שונים מאפס.
*   **חסרונות:**
    *   **לא מתאים לכל בעיה:** לא תמיד יעיל במיוחד כאשר יש קורלציה גבוהה בין התכונות.
*   **דוגמת קוד (Python):**

    ```python
    from sklearn.linear_model import Lasso
    from sklearn.model_selection import train_test_split
    import numpy as np

    # יצירת נתונים לדוגמה
    X = np.random.rand(100, 10)
    y = np.random.rand(100)

    # חלוקה לסט אימון וסט בדיקה
    X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

    # יצירת מודל Lasso עם פרמטר רגולריזציה
    lasso = Lasso(alpha=0.1)  # alpha הוא פרמטר ה-λ

    # אימון המודל
    lasso.fit(X_train, y_train)

    # חיזוי על סט הבדיקה
    y_pred = lasso.predict(X_test)

    print("Coefficients:", lasso.coef_)
    ```

2. **L2 Regularization (Ridge Regression)**

*   **עקרון:** מוסיפה "עונש" על הסכום של ריבועי המקדמים של המודל.
*   **נוסחה:**
    *   Cost Function = Loss Function + λ * Σ βᵢ² (כאשר λ הוא פרמטר הרגולריזציה, ו-βᵢ הם המקדמים)
*   **יתרונות:**
    *   **מצמצם גודל המקדמים:** מונע מצב שבו מקדם אחד שולט במודל.
    *   **יעיל כאשר יש קורלציה גבוהה בין תכונות:**  פועל טוב יותר מ-L1 כאשר יש קורלציה גבוהה בין התכונות.
*   **חסרונות:**
    *   **לא מבצע Feature Selection:** לא "הורג" מקדמים, אלא רק מצמצם אותם.
*   **דוגמת קוד (Python):**

    ```python
    from sklearn.linear_model import Ridge
    from sklearn.model_selection import train_test_split
    import numpy as np

    # יצירת נתונים לדוגמה
    X = np.random.rand(100, 10)
    y = np.random.rand(100)

    # חלוקה לסט אימון וסט בדיקה
    X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

    # יצירת מודל Ridge עם פרמטר רגולריזציה
    ridge = Ridge(alpha=0.1)  # alpha הוא פרמטר ה-λ

    # אימון המודל
    ridge.fit(X_train, y_train)

    # חיזוי על סט הבדיקה
    y_pred = ridge.predict(X_test)

    print("Coefficients:", ridge.coef_)
    ```

3. **Elastic Net Regularization**

*   **עקרון:** שילוב של L1 ו-L2 Regularization.
*   **נוסחה:**
    *   Cost Function = Loss Function + λ₁ * Σ |βᵢ| + λ₂ * Σ βᵢ²  (כאשר λ₁ ו-λ₂ הם פרמטרי הרגולריזציה עבור L1 ו-L2, בהתאמה)
*   **יתרונות:**
    *   **משלב את היתרונות של L1 ו-L2:**  מאפשר בחירת תכונות (כמו L1) תוך כדי שמירה על יציבות כאשר יש קורלציה גבוהה בין תכונות (כמו L2).
*   **חסרונות:**
    *   **מורכב יותר לכיוונון:** צריך לבחור שני פרמטרי רגולריזציה (λ₁ ו-λ₂).
*   **דוגמת קוד (Python):**

    ```python
    from sklearn.linear_model import ElasticNet
    from sklearn.model_selection import train_test_split
    import numpy as np

    # יצירת נתונים לדוגמה
    X = np.random.rand(100, 10)
    y = np.random.rand(100)

    # חלוקה לסט אימון וסט בדיקה
    X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

    # יצירת מודל ElasticNet עם פרמטרי רגולריזציה
    elastic_net = ElasticNet(alpha=0.1, l1_ratio=0.5)  # alpha = λ, l1_ratio = יחס L1

    # אימון המודל
    elastic_net.fit(X_train, y_train)

    # חיזוי על סט הבדיקה
    y_pred = elastic_net.predict(X_test)

    print("Coefficients:", elastic_net.coef_)
    ```

**בחירת פרמטר הרגולריזציה (λ):**

*   **Cross-Validation:** השיטה הנפוצה ביותר לבחירת פרמטר הרגולריזציה היא שימוש ב-Cross-Validation (למשל, K-Fold Cross-Validation).
*   **Grid Search:**  ניתן להשתמש ב-Grid Search כדי לחפש את הערך האופטימלי של λ.

**סיכום:**

| טכניקה | עקרון | יתרונות | חסרונות |
|---|---|---|---|
| L1 (Lasso) | עונש על סכום מוחלט של מקדמים | בחירת תכונות, מודלים דלילים | לא מתאים לכל בעיה |
| L2 (Ridge) | עונש על סכום ריבועי של מקדמים | מצמצם גודל מקדמים, יעיל לקורלציה | לא מבצע בחירת תכונות |
| Elastic Net | שילוב L1 ו-L2 | משלב יתרונות, גמישות | מורכב יותר לכיוונון |
