בואו נצלול לעומק של regularization בלמידת מכונה. Regularization היא טכניקה קריטית למניעת overfitting (התאמת יתר) של מודלים, במיוחד כאשר יש לנו מעט נתונים או הרבה תכונות. היא מוסיפה "קנס" לפונקציית העלות, ומעודדת מודלים פשוטים יותר.

**1. מהו Overfitting?**

Overfitting מתרחש כאשר מודל לומד את הנתונים בצורה מוגזמת, כולל רעשים ואנומליות. זה מוביל לביצועים מצוינים על נתוני האימון, אך ביצועים גרועים על נתונים חדשים (נתוני בדיקה).

**2. סוגי Regularization:**

*   **L1 Regularization (Lasso Regression):**
    *   **עיקרון:** מוסיפה לפונקציית העלות "קנס" פרופורציונלי לסכום הערכים המוחלטים של המקדמים.
    *   **נוסחה:**
        *   Cost = Loss + λ * Σ|βi|  (כאשר λ הוא פרמטר ה-regularization, ו-βi הם המקדמים)
    *   **יתרונות:**
        *   **Feature Selection:**  L1 יכול "להפיל" מקדמים לא חשובים לאפס, ובכך לבצע סוג של בחירת תכונות (feature selection).
        *   **מודל פשוט יותר:**  מייצר מודלים דלילים יותר (sparse models) עם פחות תכונות.
    *   **חסרונות:**
        *   יכול להיות רגיש לערכים קיצוניים בנתונים.
    *   **דוגמת קוד (Python, scikit-learn):**

```python
from sklearn.linear_model import Lasso
import numpy as np

# יצירת נתונים מדומים
X = np.random.rand(100, 5)
y = np.random.rand(100)

# יצירת מודל Lasso
lasso = Lasso(alpha=0.1)  # alpha הוא פרמטר ה-regularization (λ)

# אימון המודל
lasso.fit(X, y)

# הדפסת המקדמים (לצפייה באילו תכונות הפילו לאפס)
print(lasso.coef_)
```

*   **L2 Regularization (Ridge Regression):**
    *   **עיקרון:** מוסיפה לפונקציית העלות "קנס" פרופורציונלי לסכום הריבועים של המקדמים.
    *   **נוסחה:**
        *   Cost = Loss + λ * Σβi²
    *   **יתרונות:**
        *   **הפחתת גודל המקדמים:** מפחית את גודל המקדמים, אך לא מפיל אותם לאפס.
        *   **יציבות:** יציב יותר מ-L1 כאשר יש קורלציה גבוהה בין התכונות.
    *   **חסרונות:**
        *   לא מבצע feature selection.
    *   **דוגמת קוד (Python, scikit-learn):**

```python
from sklearn.linear_model import Ridge
import numpy as np

# יצירת נתונים מדומים
X = np.random.rand(100, 5)
y = np.random.rand(100)

# יצירת מודל Ridge
ridge = Ridge(alpha=0.1)  # alpha הוא פרמטר ה-regularization (λ)

# אימון המודל
ridge.fit(X, y)

# הדפסת המקדמים
print(ridge.coef_)
```

*   **Elastic Net Regularization:**
    *   **עיקרון:** שילוב של L1 ו-L2 Regularization.
    *   **נוסחה:**
        *   Cost = Loss + λ₁ * Σ|βi| + λ₂ * Σβi²
    *   **יתרונות:**
        *   משלב את היתרונות של L1 ו-L2.
        *   יכול לבצע feature selection כמו L1, תוך שמירה על יציבות כמו L2.
    *   **חסרונות:**
        *   מורכב יותר לכיוונון פרמטרים.
    *   **דוגמת קוד (Python, scikit-learn):**

```python
from sklearn.linear_model import ElasticNet
import numpy as np

# יצירת נתונים מדומים
X = np.random.rand(100, 5)
y = np.random.rand(100)

# יצירת מודל Elastic Net
elastic_net = ElasticNet(alpha=0.1, l1_ratio=0.5)  # alpha הוא פרמטר ה-regularization, l1_ratio משפיע על האיזון בין L1 ל-L2

# אימון המודל
elastic_net.fit(X, y)

# הדפסת המקדמים
print(elastic_net.coef_)
```

*   **Dropout (רשתות נוירונים):**
    *   **עיקרון:** במהלך האימון, באופן אקראי "מכבים" (מאפסים) חלק מהנוירונים בשכבה מסוימת.
    *   **יתרונות:**
        *   מונע התאמת יתר ב-neural networks.
        *   קל ליישום.
    *   **חסרונות:**
        *   דרוש כיוונון של פרמטר ה-dropout (הסיכוי לכבות נוירון).
    *   **דוגמת קוד (Python, Keras):**

```python
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import Dense, Dropout

model = Sequential()
model.add(Dense(64, activation='relu', input_shape=(10,)))
model.add(Dropout(0.5))  # 50% מהנוירונים יכובו במהלך האימון
model.add(Dense(10, activation='softmax'))

model.compile(optimizer='adam', loss='categorical_crossentropy', metrics=['accuracy'])
```

**3. בחירת פרמטר ה-Regularization (λ או alpha):**

*   **Cross-Validation:** השתמשו ב-cross-validation כדי לבחור את הערך האופטימלי של פרמטר ה-regularization.
*   **Grid Search:** בדקו מספר ערכים אפשריים של פרמטר ה-regularization, ובחרו את הערך שמניב את הביצועים הטובים ביותר על נתוני ה-validation.

**4. שיקולים נוספים:**

*   **סוג הבעיה:** בחירת סוג ה-regularization תלויה בסוג הבעיה (רגרסיה, סיווג) ובמאפייני הנתונים.
*   **נורמליזציה:** חשוב לנרמל את הנתונים לפני שימוש ב-regularization, כדי לוודא שלכל התכונות יש אותו סולם.
*   **מאזון:**  L1 ו-Elastic Net יכולים לעזור לבחור תכונות רלוונטיות, מה שיכול לשפר את הפירוש של המודל.

Regularization היא כלי חיוני בכל ארגז הכלים של מדען נתונים. הבנת העקרונות והשיטות השונות מאפשרת לכם לבנות מודלים מדויקים ויעילים יותר, ולמנוע overfitting.
