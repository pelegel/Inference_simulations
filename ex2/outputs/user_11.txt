export const getMessages = (messagesRef) => {
  return new Promise((resolve, reject) => {
    const messagesRef = database.ref('messages');
    messagesRef.on('value', (snapshot) => {
      const messages = [];
      snapshot.forEach((child) => {
        messages.push({
          id: child.key,
          text: child.val().text,
          sender: child.val().sender
        });
      });
      resolve(messages);
    });
  });
}

export const createMessage = (text) => {
  const newMessage = {
    text,
    sender: auth.currentUser?.email
  }

  return new Promise((resolve, reject) => {
    database.ref('messages').push(newMessage)
      .then(() => resolve())
      .catch((error) => reject(error));
  });
}

Prompt: תרחיב על כל סוגי ה-regularization בלמידת מכונה, כולל נוסחאות, דוגמאות קוד, ויתרונות וחסרונות

Response:
בסדר גמור, הנה הסבר מפורט על כל סוגי ה-Regularization בלמידת מכונה, כולל נוסחאות, דוגמאות קוד, ויתרונות וחסרונות:

**מהו Regularization?**

Regularization היא טכניקה המשמשת בלמידת מכונה כדי למנוע overfitting (התאמת יתר) - מצב שבו מודל לומד את נתוני האימון בצורה מושלמת מדי, אך לא מצליח להכליל נכון לנתונים חדשים. Regularization מוסיפה "עונש" למודל על מורכבות יתר, ובכך מעודדת אותו ללמוד דפוסים כלליים יותר.

**1. L1 Regularization (Lasso)**

*   **עיקרון:** מוסיפה "עונש" פרופורציונלי לסכום הערכים המוחלטים של המקדמים (coefficients) במודל.
*   **נוסחה:**
    *   פונקציית העלות (Cost Function) + λ * Σ|βᵢ|
    *   λ (Lambda) - פרמטר רגולריזציה (קבוע) ששולט בעוצמת העונש.
    *   βᵢ - המקדמים של המודל.
*   **דוגמת קוד (Python, Scikit-learn):**

```python
from sklearn.linear_model import Lasso
import numpy as np

# נתוני אימון מדומים
X = np.array([[1, 2], [3, 4], [5, 6]])
y = np.array([10, 12, 14])

# יצירת מודל Lasso
lasso = Lasso(alpha=0.1) # alpha = λ (פרמטר רגולריזציה)

# אימון המודל
lasso.fit(X, y)

# הצגת המקדמים
print(lasso.coef_)
```

*   **יתרונות:**
    *   **Feature Selection:** גורם לחלק מהמקדמים להיות בדיוק 0, מה שמאפשר לבחור רק את הפיצ'רים הרלוונטיים.
    *   **פשוט ליישום:** יחסית קל להבנה וליישום.
*   **חסרונות:**
    *   **בחירת α:**  בחירת הערך המתאים של α יכולה להיות מאתגרת.
    *   **לא מתאים למודלים עם מקדמים רבים בעלי ערכים דומים.**

**2. L2 Regularization (Ridge)**

*   **עיקרון:** מוסיפה "עונש" פרופורציונלי לסכום הריבועים של המקדמים במודל.
*   **נוסחה:**
    *   פונקציית העלות (Cost Function) + λ * Σ(βᵢ²)
    *   λ (Lambda) - פרמטר רגולריזציה.
    *   βᵢ - המקדמים של המודל.
*   **דוגמת קוד (Python, Scikit-learn):**

```python
from sklearn.linear_model import Ridge
import numpy as np

# נתוני אימון מדומים
X = np.array([[1, 2], [3, 4], [5, 6]])
y = np.array([10, 12, 14])

# יצירת מודל Ridge
ridge = Ridge(alpha=0.1) # alpha = λ (פרמטר רגולריזציה)

# אימון המודל
ridge.fit(X, y)

# הצגת המקדמים

```

**3. Elastic Net Regularization**

*   **עיקרון:** שילוב של L1 ו-L2 regularization.
*   **נוסחה:**
    *   פונקציית העלות (Cost Function) + λ₁ * Σ|βᵢ| + λ₂ * Σ(βᵢ²)
    *   λ₁ ו-λ₂ - פרמטרי רגולריזציה עבור L1 ו-L2, בהתאמה.
*   **דוגמת קוד (Python, Scikit-learn):**

```python

# הצגת המקדמים
print(elastic_net.coef_)
```

*   **חסרונות:**
    *   **בחירת שני פרמטרים:** צריך לבחור שני פרמטרי רגולריזציה (λ₁ ו-λ₂).
