**מהו Regularization?**

Regularization היא טכניקה המשמשת בלמידת מכונה כדי למנוע "התאמת יתר" (Overfitting). התאמת יתר מתרחשת כאשר המודל לומד את נתוני האימון בצורה מושלמת מדי, כולל את הרעשים והאנומליות, וכתוצאה מכך מתקשה לבצע ביצועים טובים על נתונים חדשים. Regularization מוסיפה "עונש" (penalty) למודל מורכב מדי, ומעודדת אותו ללמוד דפוסים כלליים יותר.

**סוגי Regularization:**

1. **L1 Regularization (Lasso):**

*   **עקרון:** מוסיפה עונש פרופורציונלי לסכום הערכים המוחלטים של המקדמים.
*   **נוסחה:**
    *   **פונקציית עלות:** `Cost = Loss + λ * Σ|βi|`
    *   כאשר:
        *   `Loss` - פונקציית העלות הרגילה (למשל, MSE - Mean Squared Error).
        *   `λ` (lambda) - פרמטר רגולריזציה שקובע את עוצמת העונש.
        *   `βi` - המקדמים של המודל.
*   **יתרונות:**
    *   **Feature Selection:** יכולה להפחית את מספר הפיצ'רים המשמעותיים במודל על ידי איפוס מקדמים מסוימים לאפס.
    *   **מניעת התאמת יתר:** מפחיתה מורכבות של המודל.
*   **חסרונות:**
    *   יכולה לגרום לאובדן מידע.
    *   פחות יעילה כאשר כל הפיצ'רים חשובים.
*   **דוגמת קוד (Python):**

```python
from sklearn.linear_model import Lasso
import numpy as np

# נתונים לדוגמה
X = np.array([[1, 2], [3, 4], [5, 6]])
y = np.array([7, 8, 9])

# יצירת מודל Lasso
lasso = Lasso(alpha=0.1)  # α הוא פרמטר רגולריזציה

# אימון המודל
lasso.fit(X, y)

# הדפסת המקדמים
print(lasso.coef_)
```

2. **L2 Regularization (Ridge):**

*   **עקרון:** מוסיפה עונש פרופורציונלי לסכום הריבועים של המקדמים.
*   **נוסחה:**
    *   **פונקציית עלות:** `Cost = Loss + λ * Σ(βi)^2`
    *   כאשר:
        *   `Loss` - פונקציית העלות הרגילה.
        *   `λ` (lambda) - פרמטר רגולריזציה.
        *   `βi` - המקדמים של המודל.
*   **יתרונות:**
    *   **מניעת התאמת יתר:** מפחיתה מורכבות של המודל.
    *   שומרת על כל הפיצ'רים במודל, אבל מקטינה את המשקל שלהם.
*   **חסרונות:**
    *   לא מבצעת בחירת פיצ'רים (לא מאפסת מקדמים לאפס).
*   **דוגמת קוד (Python):**

```python
from sklearn.linear_model import Ridge
import numpy as np

# נתונים לדוגמה
X = np.array([[1, 2], [3, 4], [5, 6]])
y = np.array([7, 8, 9])

# יצירת מודל Ridge
ridge = Ridge(alpha=0.1)  # α הוא פרמטר רגולריזציה

# אימון המודל
ridge.fit(X, y)

# הדפסת המקדמים
print(ridge.coef_)
```

3. **Elastic Net:**

*   **עקרון:** שילוב של L1 ו-L2 regularization.
*   **נוסחה:**
    *   **פונקציית עלות:** `Cost = Loss + λ1 * Σ|βi| + λ2 * Σ(βi)^2`
    *   `λ1` - פרמטר רגולריזציה עבור L1.
    *   `λ2` - פרמטר רגולריזציה עבור L2.
*   **יתרונות:**
    *   משלבת את היתרונות של L1 ו-L2.
    *   מבצעת בחירת פיצ'רים כמו L1, אבל גם שומרת על פיצ'רים חשובים כמו L2.
*   **חסרונות:**
    *   דורשת כוונון שני פרמטרי רגולריזציה.
*   **דוגמת קוד (Python):**

```python
from sklearn.linear_model import ElasticNet
import numpy as np

# נתונים לדוגמה
X = np.array([[1, 2], [3, 4], [5, 6]])
y = np.array([7, 8, 9])

# יצירת מודל Elastic Net
elastic_net = ElasticNet(alpha=0.1, l1_ratio=0.5)  # α ו-l1_ratio הם פרמטרי רגולריזציה

# אימון המודל
elastic_net.fit(X, y)

# הדפסת המקדמים
print(elastic_net.coef_)
```

4. **Dropout Regularization (ב-Neural Networks):**

*   **עקרון:** במהלך האימון, כל נוירון (neuron) ב-Neural Network יש סיכוי מסוים (בדרך כלל 20-50%) "להימחק" (dropped out) במהלך כל איטרציה. זה גורם לרשת ללמוד תכונות חזקות יותר שלא תלויות בנוירון ספציפי.
*   **יתרונות:**
    *   מונע התאמת יתר בצורה יעילה ברשתות נוירונים עמוקות.
    *   פשוט ליישום.
*   **חסרונות:**
    *   מאריך את זמן האימון.
*   **דוגמת קוד (Keras):**

```python
from tensorflow import keras
from tensorflow.keras.layers import Dense, Dropout

model = keras.Sequential([
    Dense(64, activation='relu', input_shape=(10,)),
    Dropout(0.3),  # 30% של הנוירונים "נמחקים" בכל איטרציה
    Dense(64, activation='relu'),
    Dense(10, activation='softmax')
])
```

**בחירת סוג ה-Regularization:**

*   **L1 (Lasso):** מתאים כאשר יש חשד שחלק גדול מהפיצ'רים אינם רלוונטיים.
*   **L2 (Ridge):** מתאים כאשר כל הפיצ'רים חשובים, אך רוצים למנוע התאמת יתר.
*   **Elastic Net:** מתאים כאשר לא בטוח איזה סוג רגולריזציה עדיף, או כאשר יש קורלציה גבוהה בין הפיצ'רים.
*   **Dropout:** מתאים לרשתות נוירונים עמוקות.

**הערות חשובות:**

*   **כוונון פרמטרים:**  הבחירה של פרמטר הרגולריזציה (λ, alpha, l1_ratio) היא קריטית. יש להשתמש בטכניקות כמו Cross-Validation כדי למצוא את הערך האופטימלי.
*   **סקלינג פיצ'רים:**  חשוב לבצע סקלינג (Scaling) של הפיצ'רים לפני השימוש ב-L1 או L2 regularization, כדי למנוע מפיצ'רים עם סדרי גודל גדולים להשתלט על התהליך.
