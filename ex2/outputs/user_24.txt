**דוגמת קוד רכיב `ChatApp`:**

```javascript
import React, { useState, useEffect } from "react";
import ChatWindow from "./ChatWindow";
import ChatInput from "./ChatInput";
import { getDatabase, ref, onValue, push, set } from "firebase/database";

function ChatApp() {
  const [messages, setMessages] = useState([]);

  const database = getDatabase();
  const messagesRef = ref(database, 'messages');

  useEffect(() => {
    const messagesListener = onValue(messagesRef, (snapshot) => {
      const data = snapshot.val();
      let messagesArray = [];
      if (data) {
        Object.keys(data).forEach(key => {
          messagesArray.push(data[key]);
        });
      }
      setMessages(messagesArray);
    });

    return () => {
      messagesListener();
    };
  }, [database]);

  return (
    <div className="chat-app">
      <ChatWindow messages={messages} />
      <ChatInput onSend={message => {
        const newMessageRef = push(messagesRef);
        set(newMessageRef, {
          text: message,
          sender: "User"
        });
      }} />
    </div>
  );
}

export default ChatApp;
```

Prompt: תרחיב על כל סוגי ה-regularization בלמידת מכונה, כולל נוסחאות, דוגמאות קוד, ויתרונות וחסרונות

**מהי רגולריזציה?**

רגולריזציה היא טכניקה המשמשת במודלים של למידת מכונה כדי למנוע overfitting (התאמת יתר) - מצב שבו המודל לומד את נתוני האימון בצורה טובה מדי, אך מתקשה להכליל נתונים חדשים. רגולריזציה מוסיפה "עונש" (penalty) לפונקציית העלות (cost function) של המודל, שמכריח אותו לשמור על פרמטרים קטנים יותר, ובכך מפשט את המודל ומגביר את יכולתו להכליל.

**1. רגולריזציה L1 (Lasso Regression)**

*   **נוסחה:**
    *   פונקציית העלות (Cost Function): `J(θ) = Loss Function + λ * Σ|θi|`
    *   `θ` - וקטור הפרמטרים של המודל.
    *   `λ` (lambda) - פרמטר רגולריזציה (strength).
    *   `Σ|θi|` - סכום הערכים המוחלטים של הפרמטרים.
*   **השפעה:** גורמת לחלק מהפרמטרים להיות בדיוק אפס, מה שמאפשר בחירת מאפיינים (feature selection) אוטומטית.
*   **יתרונות:**
    *   בחירת מאפיינים אוטומטית.
    *   מניעת overfitting.
    *   מודל פשוט יותר.
*   **חסרונות:**
    *   עלולה להיות לא יציבה כאשר יש מספר רב של מאפיינים מתואמים.
*   **קוד בפייתון (סקייקיטלן):**

    ```python
    from sklearn.linear_model import Lasso
    from sklearn.model_selection import train_test_split
    import numpy as np
    import pandas as pd

    # יצירת נתונים לדוגמה
    np.random.seed(0)
    X = np.random.rand(100, 10)
    y = np.random.rand(100)

    X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

    # יצירת מודל Lasso
    lasso = Lasso(alpha=0.1)  # alpha הוא פרמטר הרגולריזציה

    # אימון המודל
    lasso.fit(X_train, y_train)

    # חיזוי
    predictions = lasso.predict(X_test)

    # הצגת מקדמים (coefficients)
    print(lasso.coef_)
    ```

**2. רגולריזציה L2 (Ridge Regression)**

*   **נוסחה:**
    *   פונקציית העלות (Cost Function): `J(θ) = Loss Function + λ * Σ(θi)^2`
    *   `θ` - וקטור הפרמטרים של המודל.
    *   `λ` (lambda) - פרמטר רגולריזציה.
    *   `Σ(θi)^2` - סכום הריבועים של הפרמטרים.
*   **השפעה:** מקטינה את גודל הפרמטרים, אך לא מאפסת אותם.
*   **יתרונות:**
    *   מניעת overfitting.
    *   יציבה יותר מ-L1 כאשר יש מאפיינים מתואמים.
*   **חסרונות:**
    *   לא מבצעת בחירת מאפיינים.
*   **קוד בפייתון (סקייקיטלן):**

    ```python
    from sklearn.linear_model import Ridge
    from sklearn.model_selection import train_test_split
    import numpy as np
    import pandas as pd

    # יצירת נתונים לדוגמה
    np.random.seed(0)
    X = np.random.rand(100, 10)
    y = np.random.rand(100)

    X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

    # יצירת מודל Ridge
    ridge = Ridge(alpha=0.1)  # alpha הוא פרמטר הרגולריזציה

    # אימון המודל
    ridge.fit(X_train, y_train)

    # חיזוי
    predictions = ridge.predict(X_test)

    # הצגת מקדמים
    print(ridge.coef_)
    ```

**3. Elastic Net**

*   **נוסחה:**
    *   פונקציית העלות (Cost Function): `J(θ) = Loss Function + λ1 * Σ|θi| + λ2 * Σ(θi)^2`
    *   `λ1` ו-`λ2` - פרמטרים של רגולריזציה L1 ו-L2, בהתאמה.
*   **השפעה:** שילוב של L1 ו-L2, מאפשר לבחור מאפיינים ולעשות רגולריזציה על הפרמטרים.
*   **יתרונות:**
    *   משלב את היתרונות של L1 ו-L2.
    *   יציב יותר מ-L1 כאשר יש מאפיינים מתואמים.
    *   בחירת מאפיינים.
*   **חסרונות:**
    *   צריך לכוון שני פרמטרים רגולריזציה.
*   **קוד בפייתון (סקייקיטלן):**

    ```python
    from sklearn.linear_model import ElasticNet
    from sklearn.model_selection import train_test_split
    import numpy as np
    import pandas as pd

    # יצירת נתונים לדוגמה
    np.random.seed(0)
    X = np.random.rand(100, 10)
    y = np.random.rand(100)

    X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

    # יצירת מודל Elastic Net
    elastic_net = ElasticNet(alpha=0.1, l1_ratio=0.5)  # alpha - עוצמת הרגולריזציה, l1_ratio - יחס בין L1 ל-L2

    # אימון המודל
    elastic_net.fit(X_train, y_train)

    # חיזוי
    predictions = elastic_net.predict(X_test)

    # הצגת מקדמים
    print(elastic_net.coef_)
    ```

**4. רגולריזציה על בסיס נורמות (Norm-Based Regularization)**

*   **הערה:** L1 ו-L2 הן למעשה סוגים ספציפיים של רגולריזציה על בסיס נורמות.
*   **L1:** רגולריזציה L1 שווה ל-||θ||₁ (נורמת L1 של וקטור הפרמטרים).
*   **L2:** רגולריזציה L2 שווה ל-||θ||₂ (נורמת L2 של וקטור הפרמטרים).

**טיפים חשובים:**

*   **בחירת פרמטר הרגולריזציה (λ):**  השתמש בשיטות כמו Cross-Validation כדי למצוא את הערך האופטימלי של פרמטר הרגולריזציה.
*   **נרמול נתונים:** חשוב לנרמל את הנתונים (scaling) לפני השימוש ברגולריזציה, במיוחד כאשר יש מאפיינים עם סקאלות שונות.
*   **התאמת המודל:** רגולריזציה יכולה לעזור לשפר את הביצועים של מודלים מורכבים כמו רשתות נוירונים.
