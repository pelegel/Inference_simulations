**1. L1 Regularization (Lasso Regression)**

*   **עקרון:** מוסיפה ל-loss function איבר פרופורציונלי לסכום הערכים המוחלטים של המקדמים (weights) במודל.
*   **נוסחה:**
    *   `Loss = MSE(y_pred, y) + λ * Σ|w_i|`
    *   כאשר:
        *   `MSE` - Mean Squared Error (שגיאת ריבועיות ממוצעת)
        *   `λ` (lambda) - פרמטר regularization (שקובע את עוצמת ה-regularization)
        *   `w_i` - המקדמים של המודל.
*   **דוגמת קוד (Python, scikit-learn):**
    ```python
    from sklearn.linear_model import Lasso
    from sklearn.model_selection import train_test_split
    import numpy as np

    # נתונים לדוגמה
    X = np.random.rand(100, 10)
    y = np.random.rand(100)

    X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

    # יצירת מודל Lasso
    lasso = Lasso(alpha=0.1)  # alpha = λ
    lasso.fit(X_train, y_train)

    print("מקואמיצים:")
    print(lasso.coef_)
    ```
*   **יתרונות:**
    *   **Feature Selection:** L1 regularization נוטה להביא חלק מהמקדמים בדיוק לאפס, מה שגורם לאותם פיצ'רים להעלם מהמודל - זהו סוג של אוטומטי של בחירת פיצ'רים.
    *   **מודל דליל (Sparse Model):** יוצר מודלים עם מספר קטן של פיצ'רים משמעותיים.
*   **חסרונות:**
    *   **בחירה לא אופטימלית:** יכולה להסיר פיצ'רים שימושיים אם ה-lambda גדול מדי.
    *   **פחות מתאים לפיצ'רים קורלטיביים:** אם יש פיצ'רים דומים, L1 עשויה לבחור רק אחד מהם.

**2. L2 Regularization (Ridge Regression)**

*   **עקרון:** מוסיפה ל-loss function איבר פרופורציונלי לסכום הריבועים של המקדמים.
*   **נוסחה:**
    *   `Loss = MSE(y_pred, y) + λ * Σ(w_i)^2`
    *   כאשר:
        *   `MSE` - Mean Squared Error
        *   `λ` - פרמטר regularization
        *   `w_i` - המקדמים של המודל.
*   **דוגמת קוד (Python, scikit-learn):**
    ```python
    from sklearn.linear_model import Ridge
    from sklearn.model_selection import train_test_split

    X = np.random.rand(100, 10)
    y = np.random.rand(100)

    X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

    # יצירת מודל Ridge
    ridge = Ridge(alpha=0.1)  # alpha = λ
    ridge.fit(X_train, y_train)

    print("מקואמיצים:")
    print(ridge.coef_)
    ```
*   **יתרונות:**
    *   **מפחית גודל מקואמיצים:** גורם למקדמים להיות קטנים יותר, אך לא בהכרח אפס.
    *   **מתאים לפיצ'רים קורלטיביים:** עובד טוב יותר עם פיצ'רים קורלטיביים בהשוואה ל-L1.
*   **חסרונות:**
    *   **לא מבצע feature selection:** לא מסיר פיצ'רים מהמודל באופן ישיר.
    *   **יכול להיות פחות יעיל מבחינת sparsity:** לא מייצר מודלים דלילים כמו L1.

**3. Elastic Net Regularization**

*   **עקרון:** שילוב של L1 ו-L2 regularization.
*   **נוסחה:**
    *   `Loss = MSE(y_pred, y) + λ1 * Σ|w_i| + λ2 * Σ(w_i)^2`
    *   כאשר:
        *   `λ1` - פרמטר regularization עבור L1
        *   `λ2` - פרמטר regularization עבור L2
*   **דוגמת קוד (Python, scikit-learn):**
    ```python
    from sklearn.linear_model import ElasticNet
    from sklearn.model_selection import train_test_split
    import numpy as np

    # נתונים לדוגמה
    X = np.random.rand(100, 10)
    y = np.random.rand(100)

    X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

    # יצירת מודל ElasticNet
    elasticnet = ElasticNet(alpha=0.1, l1_ratio=0.5)  # alpha = λ, l1_ratio = שילוב בין L1 ו-L2
    elasticnet.fit(X_train, y_train)

    print("מקואמיצים:")
    print(elasticnet.coef_)
    ```
*   **יתרונות:**
    *   **משלב את היתרונות של L1 ו-L2:** יכול לבצע feature selection (כמו L1) תוך שמירה על ביצועים טובים עם פיצ'רים קורלטיביים (כמו L2).
    *   **גמישות:** ניתן לכוון את האיזון בין L1 ו-L2.
*   **חסרונות:**
    *   **מורכבות:** דורש כיוון של שני פרמטרים regularization.

**4. Dropout Regularization (ב-Neural Networks)**

*   **עקרון:** במהלך האימון, באופן אקראי "מכבים" נוירונים (neurons) מסוימים בשכבות הנסתרות של הרשת הנוירונית. זה מונע מהרשת להסתמך על נוירונים ספציפיים, ומעודד אותה ללמוד ייצוגים חזקים יותר.

*   **יתרונות:**
    *   **פשוט ליישום:** ניתן להוסיף בקלות לרשתות נוירוניות.

*   **חסרונות:**
    *   **עלול להאריך את זמן האימון:** מכיוון שהרשת מאמנת עם ארכיטקטורות שונות בכל איטרציה.

**5. Early Stopping**

*   **עקרון:** עוצרים את האימון כאשר הביצועים על סט האימות (validation set) מתחילים להידרדר.
