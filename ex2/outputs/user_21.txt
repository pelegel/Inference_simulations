## Regularization בלמידת מכונה: מדריך מפורט

Regularization היא טכניקה חיונית בלמידת מכונה שמטרתה למנוע overfitting - מצב שבו מודל לומד את נתוני האימון בצורה טובה מדי, אך לא מצליח להכליל היטב לנתונים חדשים. היא מוסיפה "עונש" למודל על מורכבותו, ובכך מעודדת מודלים פשוטים יותר שמתיישרים טוב יותר עם הנתונים הכלליים.

**1. L1 Regularization (Lasso Regression)**

*   **עיקרון:** מוסיפה עונש פרופורציונלי לסכום הערכים המוחלטים של המקדמים של המודל.
*   **נוסחה:**
    *   Loss function = Loss (למשל, MSE) + λ * Σ|βi|
    *   λ (למדה) הוא פרמטר regularization, ששולט בעוצמת העונש. βi הוא המקדם ה-i.
*   **דוגמת קוד (Python - Scikit-learn):**
    ```python
    from sklearn.linear_model import Lasso
    import numpy as np

    # דוגמה פשוטה עם נתונים סינטטיים
    X = np.random.rand(100, 5)
    y = np.random.rand(100)

    # יצירת מודל Lasso
    lasso = Lasso(alpha=0.1)  # alpha = λ
    # אימון המודל
    lasso.fit(X, y)

    print("המקדמים:", lasso.coef_) # חלק מהמקדמים יהיו 0 (sparse model)
    ```
*   **יתרונות:**
    *   **Sparse Features:** מוביל למודלים עם מקדמים רבים השווים לאפס, מה שמסייע לבחור את הפיצ'רים הרלוונטיים ביותר ולפשט את המודל.
    *   **Feature Selection:** יכול לשמש כשיטת feature selection אוטומטית.
*   **חסרונות:**
    *   ייתכן שהוא יגרום ל"התעלמות" מפיצ'רים חשובים אם λ גבוה מדי.
    *   פחות יעיל במקרים בהם יש קורלציה גבוהה בין הפיצ'רים.

**2. L2 Regularization (Ridge Regression)**

*   **עיקרון:** מוסיפה עונש פרופורציונלי לסכום הריבועים של המקדמים של המודל.
*   **נוסחה:**
    *   Loss function = Loss + λ * Σ(βi)^2
*   **דוגמת קוד (Python - Scikit-learn):**
    ```python
    from sklearn.linear_model import Ridge
    import numpy as np

    # דוגמה פשוטה עם נתונים סינטטיים
    X = np.random.rand(100, 5)
    y = np.random.rand(100)

    # יצירת מודל Ridge
    ridge = Ridge(alpha=0.1)  # alpha = λ
    # אימון המודל
    ridge.fit(X, y)

    print("המקדמים:", ridge.coef_) # המקדמים יהיו קטנים, אך לא בהכרח 0
    ```
*   **יתרונות:**
    *   **מתמודד עם Multicollinearity:** יעיל במקרים בהם יש קורלציה גבוהה בין הפיצ'רים (multicollinearity).
    *   **מונע Overfitting:** עוזר למנוע overfitting על ידי הפחתת גודל המקדמים.
*   **חסרונות:**
    *   לא מוביל למודלים עם מקדמים רבים השווים לאפס (לא מבצע feature selection).
    *   פחות יעיל במקרים בהם צריך לבחור מספר קטן של פיצ'רים רלוונטיים.

**3. Elastic Net Regularization**

*   **עיקרון:** שילוב של L1 ו-L2 regularization. משלב את היתרונות של שתי הטכניקות.
*   **נוסחה:**
    *   Loss function = Loss + λ1 * Σ|βi| + λ2 * Σ(βi)^2
    *   λ1 ו-λ2 הם פרמטר regularization ששולטים בעוצמת העונש של L1 ו-L2, בהתאמה.
*   **דוגמת קוד (Python - Scikit-learn):**
    ```python
    from sklearn.linear_model import ElasticNet
    import numpy as np

    # דוגמה פשוטה עם נתונים סינטטיים
    X = np.random.rand(100, 5)
    y = np.random.rand(100)

    # יצירת מודל ElasticNet
    elastic_net = ElasticNet(alpha=0.1, l1_ratio=0.5) # alpha = λ1, l1_ratio קובע את החלק של L1
    # אימון המודל
    elastic_net.fit(X, y)

    print("המקדמים:", elastic_net.coef_)
    ```
*   **יתרונות:**
    *   **משלב את היתרונות של L1 ו-L2:** מספק איזון טוב בין feature selection (L1) לטיפול ב-multicollinearity (L2).
    *   **גמיש יותר:** מאפשר שליטה על האיזון בין L1 ו-L2 באמצעות פרמטר `l1_ratio`.
*   **חסרונות:**
    *   מורכב יותר לכוון (צריך לכוונן שני פרמטרים regularization).

**4. Dropout Regularization (עבור רשתות נוירונים)**

*   **עיקרון:** בזמן האימון, בוחר באופן אקראי חלק מהנוירונים ברשת ו"מכבה" אותם. זה מונע מהרשת להסתמך על נוירונים ספציפיים ומעודד אותה ללמוד ייצוגים חזקים יותר.
*   **יישום:** מתבצע על ידי הוספת שכבת Dropout לאחר שכבות Fully Connected או Convolutional.
*   **יתרונות:**
    *   **מונע Overfitting:** יעיל מאוד ברשתות נוירונים עמוקות.
    *   **פשוט ליישום:** קל להוסיף שכבות Dropout לרשת נוירונים.
*   **חסרונות:**
    *   דורש התאמה של שיעור ה-Dropout (probability of dropping a neuron).

**5. Early Stopping**

*   **עיקרון:** עצירת האימון של המודל כאשר ביצועיו על סט ה-validation מתחילים לרדת.
*   **יישום:** ניטור הביצועים של המודל על סט validation בכל epoch, והפסקת האימון כאשר הביצועים מתחילים לרדת.
*   **יתרונות:**
    *   **פשוט ליישום:** קל ליישום ואינו דורש פרמטרים נוספים.
    *   **מונע Overfitting:** עוזר למנוע overfitting על ידי עצירת האימון בזמן האופטימלי.
*   **חסרונות:**
    *   דורש סט validation נפרד.

**טבלה מסכמת:**

| Regularization | נוסחה | יתרונות | חסרונות | מתאים ל |
|---|---|---|---|---|
| L1 (Lasso) | Loss + λ * Σ|βi| | Sparse features, Feature selection | פיצ'רים רבים, רצון לבחור פיצ'רים רלוונטיים |
| L2 (Ridge) | Loss + λ * Σ(βi)^2 | Multicollinearity, מונע Overfitting | לא מבצע feature selection | קורלציה גבוהה בין פיצ'רים |

| Dropout | - | מונע Overfitting, פשוט ליישום | צריך להתאים שיעור Dropout | רשתות נוירונים |
| Early Stopping | - | פשוט ליישום, מונע Overfitting | דורש סט validation | כל סוג מודל |

הבחירה בשיטת regularization המתאימה תלויה במאפייני הנתונים ובמודל. נסו לשחק עם פרמטרי regularization וסט validation כדי למצוא את הביצועים הטובים ביותר.
