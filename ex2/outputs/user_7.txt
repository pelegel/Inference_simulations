```javascript
import { getDatabase, ref, push, get, child, orderByChild, query, limitToLast } from "firebase/database";

export const getMessages = (database, callback) => {
  const messagesRef = query(
    get(child(ref(database), 'messages')),
    orderByChild('timestamp'),
    limitToLast(100)
  );
  get(messagesRef).then(snapshot => {
    const messages = [];
    snapshot.forEach(childSnapshot => {
      messages.push(childSnapshot.val());
    });
    callback(messages);
  });
};

export const sendMessage = (database, sender, text) => {
  const messagesRef = ref(database, 'messages');
  const newMessage = push(messagesRef);
  push(child(newMessage, {
    sender: sender,
    text: text,
    timestamp: Date.now()
  }));
};
```

**1. מהו Regularization?**

Regularization היא טכניקה בלמידת מכונה שמטרתה למנוע "התאמת יתר" (Overfitting) - מצב שבו המודל לומד את נתוני האימון בצורה מושלמת מדי, אך מתקשה להכליל על נתונים חדשים. היא עושה זאת על ידי הוספת "קנס" לפונקציית העלות (Cost Function) שהמודל מנסה למזער, מה שמכריח אותו לשמור על פרמטרים קטנים יחסית.

**2. סוגי Regularization:**

*   **L1 Regularization (Lasso Regression):**
    *   **עיקרון:** מוסיפה לפונקציית העלות את סכום הערכים המוחלטים של הפרמטרים (L1 norm).
    *   **נוסחה:** Cost = MSE + λ * Σ|βi| (כאשר λ הוא פרמטר regularization ו-βi הם הפרמטרים של המודל).
    *   **יתרונות:**
        *   **Feature Selection:** עלולה לאפס חלק מהפרמטרים, מה שמאפשר לבחור את התכונות החשובות ביותר (Sparse Model).

*   **L2 Regularization (Ridge Regression):**
    *   **עיקרון:** מוסיפה לפונקציית העלות את סכום הריבועים של הפרמטרים (L2 norm).
    *   **נוסחה:** Cost = MSE + λ * Σβi²
    *   **יתרונות:**
        *   **מונעת התאמת יתר בצורה טובה.**

*   **Elastic Net Regularization:**
    *   **עיקרון:** שילוב של L1 ו-L2 Regularization.
    *   **נוסחה:** Cost = MSE + λ1 * Σ|βi| + λ2 * Σβi²
    *   **יתרונות:**
        *   **משלבת את היתרונות של L1 ו-L2.**
        *   **מאפשרת Feature Selection (כמו L1) וגם מונעת התאמת יתר (כמו L2).**
    *   **חסרונות:**
        *   **מורכבת יותר לחישוב.**
        *   **צריך לכוון שני פרמטרים (λ1 ו-λ2).**
    *   **דוגמת קוד (Python, Scikit-Learn):**

```python

import numpy as np

# יצירת נתונים לדוגמה
```

*   **Cross-Validation:** משתמשים ב-Cross-Validation (למשל, K-Fold Cross-Validation) כדי למצוא את הערך האופטימלי של λ, שממזער את השגיאה על נתוני האימות.

**4. דוגמאות נוספות ל-Regularization:**

*   **Dropout (Neural Networks):** טכניקה בה משביתים באופן אקראי חלק מהנוירונים במהלך האימון, כדי למנוע התאמת יתר.
*   **Early Stopping:** עוצרים את האימון לפני שהמודל מתחיל להתאמן יתר על המידה, על ידי מעקב אחר השגיאה על נתוני האימות.

**5. סיכום:**

| Regularization | נוסחה  | יתרונות | חסרונות |
|---|---|---|---|
| L1 (Lasso) | Cost = MSE + λ * Σ|βi| | Feature Selection, פשוטה | עלולה לפגוע בדיוק |
| L2 (Ridge) | Cost = MSE + λ * Σβi² | מונעת התאמת יתר, יציבה | לא מבצעת Feature Selection |
| Elastic Net | Cost = MSE + λ1 * Σ|βi| + λ2 * Σβi² | משלבת יתרונות, Feature Selection | מורכבת יותר |

*   Regularization היא כלי חשוב למניעת התאמת יתר ולשיפור הכללת המודל.
