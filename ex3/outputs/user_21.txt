## Regularization בלמידת מכונה: מדריך מפורט

Regularization היא טכניקה חשובה בלמידת מכונה המונעת overfitting (התאמת יתר) של המודל לנתוני האימון. overfitting מתרחש כאשר המודל לומד את הפרטים הקטנים ביותר בנתוני האימון ולא מסוגל לבצע הכללה טובה לנתונים חדשים. Regularization מוסיפה עונש למודל על מורכבותו, מה שמאלץ אותו ללמוד דפוסים כלליים יותר ולמנוע התאמת יתר.

הנה סקירה מפורטת של סוגי ה-Regularization הנפוצים ביותר, כולל נוסחאות, דוגמאות קוד, יתרונות וחסרונות:

**1. L1 Regularization (Lasso Regression)**

*   **תיאור:** L1 regularization מוסיפה סכום של הערכים המוחלטים של המקדמים (weights) לפונקציית העלות. זה גורם למודל לבחור רק את המאפיינים החשובים ביותר ולהקטין את הערכים המוחלטים של המקדמים הלא רלוונטיים לאפס.
*   **נוסחה:**
    ```
    Cost = MSE + λ * Σ |wᵢ|
    ```
    כאשר:
    *   MSE - Mean Squared Error (פונקציית העלות)
    *   λ - פרמטר ה-Regularization (ככל שהוא גבוה יותר, העונש על המורכבות חזק יותר)
    *   wᵢ - המקדם של המאפיין ה-i
*   **יתרונות:**
    *   ביצוע feature selection - מאפשר לבחור רק את המאפיינים החשובים ביותר.
    *   יעיל במיוחד כאשר יש מספר רב של מאפיינים לא רלוונטיים.
    *   יוצר מודל פרשני יותר.
*   **חסרונות:**
    *   פחות מדויק מ-L2 regularization במקרים מסוימים.
    *   עלול להיות לא יציב מבחינה חישובית.
*   **דוגמת קוד (Python - Scikit-learn):**
    ```python
    from sklearn.linear_model import Lasso
    from sklearn.datasets import make_regression
    import numpy as np

    X, y = make_regression(n_samples=100, n_features=10, random_state=42)

    lasso = Lasso(alpha=1.0)  # α הוא פרמטר ה-Regularization (λ)
    lasso.fit(X, y)
    print(lasso.coef_)
    ```

**2. L2 Regularization (Ridge Regression)**

*   **תיאור:** L2 regularization מוסיפה סכום של ריבועי המקדמים לפונקציית העלות. זה גורם למודל להקטין את כל המקדמים, אך לא לאפס אותם.
*   **נוסחה:**
    ```
    Cost = MSE + λ * Σ wᵢ²
    ```
    כאשר:
    *   MSE - Mean Squared Error (פונקציית העלות)
    *   λ - פרמטר ה-Regularization (ככל שהוא גבוה יותר, העונש על המורכבות חזק יותר)
    *   wᵢ - המקדם של המאפיין ה-i
*   **יתרונות:**
    *   מפחיתת את overfitting בצורה יעילה.
    *   יציבה מבחינה חישובית.
    *   משפרת את הכלליות של המודל.
*   **חסרונות:**
    *   לא מבצעת feature selection - כל המאפיינים נשארים במודל.
    *   פחות יעילה מ-L1 regularization במקרים מסוימים.
*   **דוגמת קוד (Python - Scikit-learn):**
    ```python
    from sklearn.linear_model import Ridge
    from sklearn.datasets import make_regression
    import numpy as np

    X, y = make_regression(n_samples=100, n_features=10, random_state=42)

    ridge = Ridge(alpha=1.0)  # α הוא פרמטר ה-Regularization (λ)
    ridge.fit(X, y)
    print(ridge.coef_)
    ```

**3. Elastic Net Regularization**

*   **תיאור:** Elastic Net regularization היא שילוב של L1 ו-L2 regularization. היא מוסיפה סכום של הערכים המוחלטים וריבועי המקדמים לפונקציית העלות. זה משלב את היתרונות של שתי הטכניקות ומאפשר לבצע feature selection תוך שמירה על דיוק גבוה.
*   **נוסחה:**
    ```
    Cost = MSE + λ₁ * Σ |wᵢ| + λ₂ * Σ wᵢ²
    ```
    כאשר:
    *   MSE - Mean Squared Error (פונקציית העלות)
    *   λ₁ - פרמטר ה-Regularization עבור L1
    *   λ₂ - פרמטר ה-Regularization עבור L2
*   **יתרונות:**
    *   משלב את היתרונות של L1 ו-L2 regularization.
    *   מאפשר לבצע feature selection בצורה יעילה.
    *   משפר את הכלליות של המודל.
*   **חסרונות:**
    *   מורכב יותר מ-L1 ו-L2 regularization.
    *   דורש כוונון של שני פרמטרים (λ₁ ו-λ₂).
*   **דוגמת קוד (Python - Scikit-learn):**
    ```python
    from sklearn.linear_model import ElasticNet
    from sklearn.datasets import make_regression
    import numpy as np

    X, y = make_regression(n_samples=100, n_features=10, random_state=42)

    elastic_net = ElasticNet(alpha=1.0, l1_ratio=0.5)  # α הוא פרמטר ה-Regularization (λ), l1_ratio שולט על היחס בין L1 ו-L2
    elastic_net.fit(X, y)
    print(elastic_net.coef_)
    ```

**4. Dropout Regularization (לרוב משמש ברשתות נוירונים)**

*   **תיאור:** Dropout הוא טכניקה שמשמשת ברשתות נוירונים כדי למנוע overfitting. במהלך האימון, יחידות נוירונים מסוימות מושמטות באופן אקראי מהרשת. זה גורם לרשת ללמוד תכונות יותר כלליות ולהפחית את התלות שלה ביחידות נוירונים מסוימות.
*   **הסבר:** בכל איטרציה של האימון, כל נוירון מוחזר בסיכוי של `p`. היתר מושמט. במהלך הבדיקה (inference), כל הנוירונים פעילים.
*   **יתרונות:**
    *   מפחיתת overfitting בצורה יעילה.
    *   פשוטה ליישום.
    *   שימושית במיוחד ברשתות נוירונים עמוקות.
*   **חסרונות:**
    *   דורשת כוונון של פרמטר ה-Dropout (p).
    *   עלולה להאריך את זמן האימון.
*   **דוגמת קוד (Python - TensorFlow/Keras):**
    ```python
    import tensorflow as tf
    from tensorflow.keras.layers import Dense, Dropout

    model = tf.keras.Sequential([
    ```

**הערות חשובות:**

*   בחירת סוג ה-Regularization המתאים תלויה במאפיינים של הנתונים ובסוג המודל.
*   פרמטר ה-Regularization (λ או α) צריך להיות מותאם באמצעות cross-validation כדי למצוא את הערך האופטימלי.
*   ניתן להשתמש בטכניקות שונות של Regularization בשילוב כדי להשיג ביצועים טובים יותר.
