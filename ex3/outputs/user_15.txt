        variant="outlined"

      />

## רגולריזציה בלמידת מכונה: מדריך מפורט

רגולריזציה (Regularization) היא טכניקה חשובה בלמידת מכונה שנועדה למנוע overfitting - מצב בו מודל לומד את הנתונים בצורה מוגזמת, כולל רעשים ופרטים לא רלוונטיים, ומתקשה להכליל לנתונים חדשים. רגולריזציה מוסיפה עונש למודל על מורכבותו, ובכך מעודדת אותו ללמוד מודלים פשוטים יותר שיש להם יכולת הכללה טובה יותר. הנה סקירה מפורטת של סוגי הרגולריזציה הנפוצים ביותר:

**1. L1 Regularization (Lasso)**

*   **מהי:** L1 רגולריזציה מוסיפה סכום הערכים המוחלטים של המשקלים במודל לפונקציית העלות.
*   **נוסחה:** `Cost = Loss + λ * Σ |wᵢ|`, כאשר:
    *   `Loss` היא פונקציית העלות של המודל (למשל, MSE).
    *   `λ` הוא פרמטר הרגולריזציה שקובע את העוצמה של העונש על המורכבות.
    *   `wᵢ` הוא המשקל של התכונה ה-i.
*   **דוגמת קוד (Python):**

```python
import numpy as np
from sklearn.linear_model import Lasso
from sklearn.datasets import make_classification

X, y = make_classification(n_samples=100, n_features=10, random_state=42)

lasso = Lasso(alpha=0.1) # alpha הוא פרמטר הרגולריזציה (λ)
lasso.fit(X, y)

print("משקלים:", lasso.coef_)
```

*   **יתרונות:**
    *   **בחירת תכונות:** L1 רגולריזציה יכולה לגרום למשקלים של תכונות לא רלוונטיות להיות שווים לאפס, ובכך לבחור רק את התכונות הרלוונטיות ביותר.
    *   **פשטות:** עוזרת ליצור מודלים פשוטים יותר.
*   **חסרונות:**
    *   **לא מתאים לבעיות עם הרבה משקלים קטנים:** L1 רגולריזציה עשויה להיות פחות יעילה בבעיות שבהן יש הרבה משקלים קטנים שכולם תורמים לפתרון.
    *   **רגישות לערכי קיצון:** רגישה לערכי קיצון בנתונים.

**2. L2 Regularization (Ridge)**

*   **מהי:** L2 רגולריזציה מוסיפה סכום ריבועי המשקלים במודל לפונקציית העלות.
*   **נוסחה:** `Cost = Loss + λ * Σ wᵢ²`, כאשר:
    *   `Loss` היא פונקציית העלות של המודל.
    *   `λ` הוא פרמטר הרגולריזציה.
    *   `wᵢ` הוא המשקל של התכונה ה-i.
*   **דוגמת קוד (Python):**

```python
import numpy as np
from sklearn.linear_model import Ridge
from sklearn.datasets import make_classification

X, y = make_classification(n_samples=100, n_features=10, random_state=42)

ridge = Ridge(alpha=0.1) # alpha הוא פרמטר הרגולריזציה (λ)
ridge.fit(X, y)

print("משקלים:", ridge.coef_)
```

*   **יתרונות:**
    *   **יציבות:** יציבה יותר מ-L1 רגולריזציה.
    *   **מפחיתת את השפעת ערכי קיצון:** מפחיתה את השפעת ערכי קיצון בנתונים.
*   **חסרונות:**
    *   **לא מבצעת בחירת תכונות:** לא גורמת למשקלים של תכונות לא רלוונטיות להיות שווים לאפס.
    *   **פחות יעילה בבחירת תכונות:** לא יעילה כמו L1 רגולריזציה בבחירת תכונות.

**3. Elastic Net**

*   **מהי:** שילוב של L1 ו-L2 רגולריזציה. מוסיפה סכום של הערכים המוחלטים וסכום ריבועי המשקלים לפונקציית העלות.
*   **נוסחה:** `Cost = Loss + λ₁ * Σ |wᵢ| + λ₂ * Σ wᵢ²`, כאשר:
    *   `Loss` היא פונקציית העלות של המודל.
    *   `λ₁` הוא פרמטר הרגולריזציה של L1.
    *   `λ₂` הוא פרמטר הרגולריזציה של L2.
    *   `wᵢ` הוא המשקל של התכונה ה-i.
*   **דוגמת קוד (Python):**

```python
import numpy as np
from sklearn.linear_model import ElasticNet
from sklearn.datasets import make_classification

X, y = make_classification(n_samples=100, n_features=10, random_state=42)

elastic_net = ElasticNet(alpha=0.1, l1_ratio=0.5) # alpha הוא פרמטר הרגולריזציה הכולל, l1_ratio קובע את היחס בין L1 ל-L2
elastic_net.fit(X, y)

print("משקלים:", elastic_net.coef_)
```

*   **יתרונות:**
    *   **שילוב של היתרונות:** משלבת את היתרונות של L1 ו-L2 רגולריזציה.
    *   **בחירת תכונות ויציבות:** מאפשרת בחירת תכונות תוך שמירה על יציבות המודל.
*   **חסרונות:**
    *   **מורכבות:** מורכבת יותר לכוון מכיוון שיש שני פרמטרים לרגולריזציה.

**4. Dropout (במיוחד ברשתות נוירונים)**

*   **מהי:** טכניקה שמשמשת בעיקר ברשתות נוירונים. במהלך האימון, בצורה אקראית, משביתה נוירונים. זה כופה על הרשת ללמוד תכונות חזקות יותר שאינן תלויות בנוירונים ספציפיים.
*   **איך זה עובד:** עבור כל נוירון, יש הסתברות שנקבעת מראש שהנוירון יושבת במהלך האימון. הסתברות זו נקראת "dropout rate".
*   **יתרונות:**
    *   **מניעת overfitting:** עוזרת למנוע overfitting על ידי כפיית הרשת ללמוד תכונות חזקות יותר.
    *   **שיפור הכללה:** משפרת את היכולת הכללית של הרשת.
*   **חסרונות:**
    *   **צריך להתאים את הרשת:** צריך להתאים את הרשת כך שהיא תתמודד עם היעדר נוירונים אקראי במהלך האימון.

**הערות חשובות:**

*   **בחירת הפרמטרים:** בחירת הפרמטרים של הרגולריזציה (λ, α, l1_ratio) היא קריטית להצלחת הטכניקה. ניתן להשתמש בטכניקות כמו cross-validation כדי למצוא את הערכים האופטימליים.
*   **נורמליזציה:** לפני הפעלת רגולריזציה, מומלץ לנרמל את הנתונים כדי למנוע השפעה של סקאלה שונה של התכונות.

רגולריזציה היא כלי חיוני בלמידת מכונה שיכול לשפר את הביצועים והיכולת הכללית של המודלים שלך. הבנת הסוגים השונים של רגולריזציה והאופן שבו הם עובדים תסייע לך לבחור את הטכניקה המתאימה ביותר לבעיה הספציפית שלך.
