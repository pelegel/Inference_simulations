## רגולריזציה בלמידת מכונה: מדריך מקיף

רגולריזציה היא טכניקה חיונית בלמידת מכונה שמטרתה למנוע התאמת יתר (overfitting) של המודל לנתוני האימון. התאמת יתר מתרחשת כאשר המודל לומד את הרעש בנתוני האימון במקום את הקשרים האמיתיים בין המשתנים. זה גורם למודל להצליח טוב על נתוני האימון אך להיכשל על נתונים חדשים. רגולריזציה מוסיפה מידע נוסף ללמידה כדי למנוע התאמת יתר.

הנה פירוט מקיף של סוגי הרגולריזציה השונים:

**1. L1 רגולריזציה (Lasso Regression)**

*   **תיאוריה:** L1 רגולריזציה מוסיפה סכום של הערך המוחלט של משקלי המודל לפונקציית העלות. זה מעודד את המודל לקבוע חלק מהמשקלים לאפס, ובכך לבחור רק את התכונות החשובות ביותר.
*   **נוסחה:**
    ```
    Cost = MSE + λ * Σ |w_i|
    ```
    *   MSE - Mean Squared Error (פונקציית העלות הסטנדרטית)
    *   λ - פרמטר הרגולריזציה (קובע את עוצמת הרגולריזציה)
    *   w_i - המשקל של תכונה i
*   **יתרונות:**
    *   בחירת תכונות באופן אוטומטי (feature selection)
    *   יעיל במיוחד במקרים שבהם יש מספר רב של תכונות
    *   נוצר מודל פשוט יותר וקל יותר לפירוש
*   **חסרונות:**
    *   יכול להיות לא יציב (במיוחד עם נתונים בעלי קוליינאריות גבוהה)
    *   דורש כוונון קפדני של פרמטר הרגולריזציה (λ)
*   **דוגמת קוד (Python):**
    ```python
    from sklearn.linear_model import Lasso
    from sklearn.datasets import make_classification

    X, y = make_classification(n_samples=100, n_features=10, random_state=42)
    lasso = Lasso(alpha=0.1)
    lasso.fit(X, y)
    print(lasso.coef_) # מציג את המשקלים של התכונות, חלקן יהיו אפס
    ```

**2. L2 רגולריזציה (Ridge Regression)**

*   **תיאוריה:** L2 רגולריזציה מוסיפה סכום של ריבועי המשקלים לפונקציית העלות. זה מעודד את המודל לקטון את המשקלים, אך לא לבטל אותם.
*   **נוסחה:**
    ```
    Cost = MSE + λ * Σ w_i^2
    ```
    *   MSE - Mean Squared Error (פונקציית העלות הסטנדרטית)
    *   λ - פרמטר הרגולריזציה (קובע את עוצמת הרגולריזציה)
    *   w_i - המשקל של תכונה i
*   **יתרונות:**
    *   יציבה יותר מ-L1 רגולריזציה
    *   פועלת היטב במקרים שבהם יש קוליינאריות גבוהה בין התכונות
    *   מפחיתה את השונות של המודל
*   **חסרונות:**
    *   לא מבצעת בחירת תכונות באופן אוטומטי
    *   פחות יעילה מ-L1 רגולריזציה במקרים שבהם יש מספר רב של תכונות לא רלוונטיות
*   **דוגמת קוד (Python):**
    ```python
    from sklearn.linear_model import Ridge
    from sklearn.datasets import make_classification

    X, y = make_classification(n_samples=100, n_features=10, random_state=42)
    ridge = Ridge(alpha=0.1)
    ridge.fit(X, y)
    print(ridge.coef_) # מציג את המשקלים של התכונות, כולם יהיו קטנים
    ```

**3. Elastic Net Regression**

*   **תיאוריה:** Elastic Net רגולריזציה היא שילוב של L1 ו-L2 רגולריזציה. היא מוסיפה סכום של הערך המוחלט וריבועי המשקלים לפונקציית העלות. זה משלב את היתרונות של שתי השיטות ומספק גמישות רבה יותר.
*   **נוסחה:**
    ```
    Cost = MSE + λ₁ * Σ |w_i| + λ₂ * Σ w_i^2
    ```
    *   MSE - Mean Squared Error (פונקציית העלות הסטנדרטית)
    *   λ₁ - פרמטר רגולריזציה עבור L1
    *   λ₂ - פרמטר רגולריזציה עבור L2
*   **יתרונות:**
    *   משלב את היתרונות של L1 ו-L2 רגולריזציה
    *   מבצעת בחירת תכונות ויעילה במקרים של קוליינאריות גבוהה
*   **חסרונות:**
    *   דורש כוונון של שני פרמטרים (λ₁ ו-λ₂)
    *   יכול להיות מורכב יותר לניתוח מאשר L1 או L2 רגולריזציה בודדת
*   **דוגמת קוד (Python):**
    ```python
    from sklearn.linear_model import ElasticNet
    from sklearn.datasets import make_classification

    X, y = make_classification(n_samples=100, n_features=10, random_state=42)
    elastic_net = ElasticNet(alpha=0.1, l1_ratio=0.5)
    elastic_net.fit(X, y)
    print(elastic_net.coef_) # מציג את המשקלים של התכונות, חלקן אפס וחלקן קטנות
    ```

**4. Dropout**

*   **יתרונות:**

    *   מפחיתה את ההתאמת יתר ביעילות
*   **חסרונות:**
    *   דורשת כוונון של פרמטר `p` (ההסתברות לשרידה של נוירון)
*   **דוגמת קוד (Python - TensorFlow/Keras):**
