## Regularization בלמידת מכונה: סוגים, נוסחאות, קוד, יתרונות וחסרונות

Regularization היא טכניקה חשובה בלמידת מכונה שמטרתה למנוע overfitting – מצב שבו מודל לומד את נתוני האימון בצורה מושלמת מדי, כך שהוא לא מצליח להכליל נכון לנתונים חדשים. Regularization מוסיפה מידע נוסף לפונקציית העלות של המודל, שמכוון אותו לבחור פרמטרים שמובילים למודל פשוט יותר שיש סיכוי טוב יותר שיכליל נכון. הנה סקירה מפורטת של סוגי ה-regularization השונים:

**1. L1 Regularization (Lasso)**

*   **תיאור:** מוסיפה לפונקציית העלות סכום של הערכים המוחלטים של הפרמטרים של המודל. מעודדת את המודל לבחור פרמטרים רבים ששווים לאפס.
*   **נוסחה:**
    ```
    Cost = Loss + λ * Σ |w_i|
    ```
    כאשר:
    *   Loss היא פונקציית העלות הרגילה של המודל.
    *   λ הוא פרמטר ה-regularization שמגדיר את עוצמת ההשפעה של ה-regularization על המודל.
    *   w_i הוא הפרמטר ה-i במודל.
*   **יתרונות:**
    *   בחירת פרמטרים: יכולה לבחור פרמטרים לא רלוונטיים ולהפוך אותם לאפס, מה שיכול לפשט את המודל ולהפוך אותו ליותר פרשני.
    *   טיפול בממדים גבוהים: יעילה במיוחד כאשר יש מספר רב של פרמטרים במודל.
*   **חסרונות:**
    *   קושי בפתרון: פונקציית העלות אינה גזירה בכל נקודה, מה שמקשה על מציאת המינימום שלה.
    *   רגישות לנתונים: יכולה להיות רגישה יותר לשינויים בנתוני האימון מאשר סוגים אחרים של regularization.
*   **קוד (Python - scikit-learn):**
    ```python
    from sklearn.linear_model import Lasso
    import numpy as np

    # נתונים לדוגמה
    X = np.array([[1], [2], [3], [4], [5]])
    y = np.array([1, 2, 3, 4, 5])

    # הגדרת מודל L1 regularization
    lasso = Lasso(alpha=0.1)  # alpha הוא פרמטר ה-regularization (λ)

    # אימון המודל
    lasso.fit(X, y)

    # הצגת הפרמטרים של המודל
    print(lasso.coef_)
    ```

**2. L2 Regularization (Ridge)**

*   **תיאור:** מוסיפה לפונקציית העלות סכום של ריבועי הפרמטרים של המודל. מעודדת את המודל לבחור פרמטרים בעלי ערכים קטנים, אך לא בהכרח אפס.
*   **נוסחה:**
    ```
    Cost = Loss + λ * Σ w_i^2
    ```
    כאשר:
    *   Loss היא פונקציית העלות הרגילה של המודל.
    *   λ הוא פרמטר ה-regularization שמגדיר את עוצמת ההשפעה של ה-regularization על המודל.
    *   w_i הוא הפרמטר ה-i במודל.
*   **יתרונות:**
    *   פשטות: פונקציית העלות גזירה בכל נקודה, מה שמקל על מציאת המינימום שלה.
    *   יציבות: פחות רגישה לשינויים בנתוני האימון מאשר L1 regularization.
*   **חסרונות:**
    *   לא מבצעת בחירת פרמטרים: לא יכולה לבחור פרמטרים לא רלוונטיים ולהפוך אותם לאפס.
*   **קוד (Python - scikit-learn):**
    ```python
    from sklearn.linear_model import Ridge
    import numpy as np

    # נתונים לדוגמה
    X = np.array([[1], [2], [3], [4], [5]])
    y = np.array([1, 2, 3, 4, 5])

    # הגדרת מודל L2 regularization
    ridge = Ridge(alpha=0.1)  # alpha הוא פרמטר ה-regularization (λ)

    # אימון המודל
    ridge.fit(X, y)

    # הצגת הפרמטרים של המודל
    print(ridge.coef_)
    ```

**3. Elastic Net Regularization**

*   **תיאור:** שילוב של L1 ו-L2 regularization. משלב את היתרונות של שתי הטכניקות.
*   **נוסחה:**
    ```
    Cost = Loss + λ1 * Σ |w_i| + λ2 * Σ w_i^2
    ```
    כאשר:
    *   Loss היא פונקציית העלות הרגילה של המודל.
    *   λ1 הוא פרמטר ה-regularization עבור L1 regularization.
    *   λ2 הוא פרמטר ה-regularization עבור L2 regularization.
    *   w_i הוא הפרמטר ה-i במודל.
*   **יתרונות:**
    *   שילוב יתרונות: משלב את היתרונות של L1 ו-L2 regularization.
    *   בחירת פרמטרים ויציבות: יכול לבחור פרמטרים לא רלוונטיים ולהפוך אותם לאפס, תוך שמירה על יציבות.
*   **חסרונות:**
    *   מורכבות: יש צורך להתאים שני פרמטרים (λ1 ו-λ2).
*   **קוד (Python - scikit-learn):**
    ```python
    from sklearn.linear_model import ElasticNet
    import numpy as np

    # נתונים לדוגמה
    X = np.array([[1], [2], [3], [4], [5]])
    y = np.array([1, 2, 3, 4, 5])

    # הגדרת מודל Elastic Net regularization
    elastic_net = ElasticNet(alpha=0.1, l1_ratio=0.5)  # alpha הוא פרמטר ה-regularization (λ)
    # l1_ratio שולט על האיזון בין L1 ו-L2 regularization

    # אימון המודל
    elastic_net.fit(X, y)

    # הצגת הפרמטרים של המודל
    print(elastic_net.coef_)
    ```

**4. Dropout Regularization (Neural Networks)**

*   **יתרונות:**
    *   מניעת overfitting: מפחיתה את הסיכון ל-overfitting.
    *   שיפור הכלליזציה: משפרת את היכולת של הרשת להכליל נכון לנתונים חדשים.
*   **חסרונות:**
    *   הגדלת זמן האימון: יכולה להאריך את זמן האימון של הרשת.
    *   התאמת פרמטרים: יש צורך להתאים את הסתברות ה-Dropout.
*   **קוד (Python - TensorFlow/Keras):**
