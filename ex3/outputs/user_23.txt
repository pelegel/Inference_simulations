## Regularization בלמידת מכונה - סקירה מפורטת

Regularization היא טכניקה שנועדה למנוע overfitting (התאמת יתר) במודלים של למידת מכונה. overfitting מתרחש כאשר המודל לומד את הנתונים בצורה מושלמת מדי, עד כדי כך שהוא לא מצליח להכליל נכון לנתונים חדשים. Regularization מוסיפה עונש למודל על מורכבותו, ובכך מעודדת אותו ללמוד פונקציות פשוטות יותר שיכולות להכליל טוב יותר.

הנה סקירה מפורטת של סוגי ה-Regularization העיקריים:

**1. L1 Regularization (Lasso Regularization)**

*   **עיקרון:** מוסיפה לסכום השגיאה עונש שפרופורציונלי לסכום הערכים המוחלטים של המשקלים במודל.
*   **נוסחה:**
    ```
    Loss = MSE + λ * Σ |w_i|
    ```
    כאשר:
    *   MSE (Mean Squared Error) - סכום השגיאה הריבועית.
    *   λ (lambda) - פרמטר ה-Regularization.
    *   w_i - המשקל של הפיצ'ר ה-i.
*   **יתרונות:**
    *   יכולת לבחור פיצ'רים: נוטה לדחוף משקלים של פיצ'רים לא רלוונטיים לאפס, ובכך לבצע בחירת פיצ'רים באופן אוטומטי.
    *   יעילה במיוחד כאשר יש מספר רב של פיצ'רים ורק חלק מהם רלוונטיים.
*   **חסרונות:**
    *   נוטה לבחור פיצ'רים באופן שרירותי אם יש קורלציה גבוהה בין הפיצ'רים.
    *   יכולה להיות לא יציבה ורגישה לערך ה-λ.
*   **דוגמה קוד (Python - scikit-learn):**
    ```python
    from sklearn.linear_model import Lasso
    from sklearn.datasets import make_regression

    # יצירת נתונים לדוגמה
    X, y = make_regression(n_samples=100, n_features=10, random_state=42)

    # יצירת מודל Lasso עם פרמטר regularization של 0.1
    lasso = Lasso(alpha=0.1)

    # אימון המודל
    lasso.fit(X, y)

    # הדפסת המשקלים של המודל
    print(lasso.coef_)
    ```

**2. L2 Regularization (Ridge Regularization)**

*   **עיקרון:** מוסיפה לסכום השגיאה עונש שפרופורציונלי לסכום הריבועים של המשקלים במודל.
*   **נוסחה:**
    ```
    Loss = MSE + λ * Σ w_i^2
    ```
    כאשר:
    *   MSE - סכום השגיאה הריבועית.
    *   λ - פרמטר ה-Regularization.
    *   w_i - המשקל של הפיצ'ר ה-i.
*   **יתרונות:**
    *   פשוטה ליישום ויעילה ברוב המקרים.
    *   יציבה יותר מ-L1 regularization.
    *   מפחיתה את השפעת פיצ'רים בעלי שונות גבוהה.
*   **חסרונות:**
    *   לא מבצעת בחירת פיצ'רים באופן אוטומטי.
    *   פחות יעילה מ-L1 regularization כאשר יש מספר רב של פיצ'רים לא רלוונטיים.
*   **דוגמה קוד (Python - scikit-learn):**
    ```python
    from sklearn.linear_model import Ridge
    from sklearn.datasets import make_regression

    # יצירת נתונים לדוגמה
    X, y = make_regression(n_samples=100, n_features=10, random_state=42)

    # יצירת מודל Ridge עם פרמטר regularization של 0.1
    ridge = Ridge(alpha=0.1)

    # אימון המודל
    ridge.fit(X, y)

    # הדפסת המשקלים של המודל
    print(ridge.coef_)
    ```

**3. Elastic Net Regularization**

*   **עיקרון:** שילוב של L1 ו-L2 regularization. מוסיפה לסכום השגיאה עונש שפרופורציונלי לסכום הערכים המוחלטים של המשקלים ולסכום הריבועים של המשקלים במודל.
*   **נוסחה:**
    ```
    Loss = MSE + λ₁ * Σ |w_i| + λ₂ * Σ w_i^2
    ```
    כאשר:
    *   MSE - סכום השגיאה הריבועית.
    *   λ₁ - פרמטר ה-L1 regularization.
    *   λ₂ - פרמטר ה-L2 regularization.
    *   w_i - המשקל של הפיצ'ר ה-i.
*   **יתרונות:**
    *   משלבת את היתרונות של L1 ו-L2 regularization.
    *   יכולה להיות יעילה במיוחד כאשר יש מספר רב של פיצ'רים וקורלציה גבוהה ביניהם.
*   **חסרונות:**
    *   מורכבת יותר ליישום מ-L1 ו-L2 regularization.
    *   יש צורך לכוון שני פרמטרים (λ₁ ו-λ₂).
*   **דוגמה קוד (Python - scikit-learn):**
    ```python
    from sklearn.linear_model import ElasticNet
    from sklearn.datasets import make_regression

    # יצירת נתונים לדוגמה
    X, y = make_regression(n_samples=100, n_features=10, random_state=42)

    # יצירת מודל Elastic Net עם פרמטרים של regularization
    elastic_net = ElasticNet(alpha=0.1, l1_ratio=0.5)

    # אימון המודל
    elastic_net.fit(X, y)

    # הדפסת המשקלים של המודל
    print(elastic_net.coef_)
    ```

**4. Dropout Regularization**

*   **עיקרון:** טכניקה שמשמשת ברשתות נוירונים. במהלך האימון, Dropout משבית באופן אקראי חלק מהנוירונים במודל. זה מונע מהנוירונים להיות תלויים מדי אחד בשני ומעודד אותם ללמוד ייצוגים עצמאיים יותר של הנתונים.
*   **יתרונות:**
    *   מפחיתה את overfitting בצורה יעילה ברשתות נוירונים.
    *   פשוטה ליישום.
*   **חסרונות:**
    *   יכולה להאריך את זמן האימון.
    *   יש צורך לכוון את שיעור ה-Dropout.
*   **דוגמה קוד (Python - TensorFlow/Keras):**

    import tensorflow as tf
